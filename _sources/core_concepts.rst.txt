Data
===============
`united` provides the :py:class:`~data.timeseries.TimeSeriesDataset` and :py:class:`~data.timeseries.SensorTimeSeriesDataset` for working with time series data much easier. :py:class:`~data.timeseries.TimeSeriesDataset` comes with a :py:meth:`~data.timeseries.TimeSeriesDataset.to_dataloader` method to convert it to a dataloader and a :py:meth:`~data.timeseries.TimeSeriesDataset.from_dataset` method to create another dataset with the same parameters except its data, the common use cases are to creating validation and test dataset from a training datasets with the same parameters and scalers.

Features:
---------

* Support multivariate input​.
* Support multiple timeseries​.
* Support missing data
    * Missing rows: forward filling last available rows​.
    * Missing values: imputers​.
* Handle unknown values for categorical features​.
* Multiple Input types: Real + categorical values​.
* Multiple Target types: Real + categorical targets​.
* Multiple model architect type
    * Encoder – Decoder​
    * Encoder only​
    * Decoder only


Models
===============

`united` provides a .from_dataset() method for each model that takes a :py:class:`~data.timeseries.TimeSeriesDataset` or :py:class:`~data.timeseries.SensorTimeSeriesDataset` and additional parameters that cannot directy derived from the dataset such as, e.g. learning_rate or hidden_size.
